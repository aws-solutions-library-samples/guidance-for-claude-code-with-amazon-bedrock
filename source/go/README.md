# credential-process (Go)

AWS credential helper that exchanges OIDC tokens for temporary AWS credentials via browser-based authentication.

Supports Okta, Auth0, Microsoft Entra ID (Azure AD), and AWS Cognito User Pool as identity providers. Federation via direct STS (AssumeRoleWithWebIdentity) or Cognito Identity Pool.

## Prerequisites

- Go 1.24+
- A configured `config.json` (generated by `ccwb init`)

## Build

```bash
# Build for your current platform
go build ./cmd/credential-process/

# Build for all platforms (outputs to bin/)
make all

# Build for a specific platform
make macos-arm64
make macos-intel
make linux-x64
make linux-arm64
make windows
```

## Install

Copy the binary and your `config.json` to `~/claude-code-with-bedrock/`:

```bash
mkdir -p ~/claude-code-with-bedrock
cp bin/credential-process-macos-arm64 ~/claude-code-with-bedrock/credential-process
cp config.json ~/claude-code-with-bedrock/
```

Then configure your AWS profile in `~/.aws/config`:

```ini
[profile ClaudeCode]
credential_process = /Users/you/claude-code-with-bedrock/credential-process --profile ClaudeCode
region = us-east-1
```

## Usage

```bash
# Normal mode (called by AWS CLI/SDK automatically)
credential-process --profile ClaudeCode

# Check if cached credentials are still valid
credential-process --check-expiration --profile ClaudeCode

# Refresh credentials if expired (for cron jobs, session storage only)
credential-process --refresh-if-needed --profile ClaudeCode

# Clear cached credentials and force re-authentication
credential-process --clear-cache --profile ClaudeCode

# Store Azure AD client secret in OS keyring
credential-process --set-client-secret --profile ClaudeCode

# Debug mode
COGNITO_AUTH_DEBUG=1 credential-process --profile ClaudeCode
```

## Configuration

The binary reads `config.json` from the same directory as the binary, or from `~/claude-code-with-bedrock/config.json`.

```json
{
  "profiles": {
    "ClaudeCode": {
      "provider_domain": "login.microsoftonline.com/tenant-id/v2.0",
      "client_id": "your-client-id",
      "provider_type": "auto",
      "aws_region": "us-east-1",
      "credential_storage": "session",
      "federation_type": "direct",
      "federated_role_arn": "arn:aws:iam::123456789012:role/ClaudeCodeRole"
    }
  }
}
```

| Field | Description |
|-------|-------------|
| `provider_domain` | OIDC provider domain |
| `client_id` | OAuth2 client ID |
| `provider_type` | `okta`, `auth0`, `azure`, `cognito`, or `auto` (detect from domain) |
| `aws_region` | AWS region for STS/Cognito calls |
| `credential_storage` | `session` (file) or `keyring` (OS secure storage) |
| `federation_type` | `direct` (STS) or `cognito` (Identity Pool) |
| `federated_role_arn` | IAM role ARN (direct mode) |
| `identity_pool_id` | Cognito Identity Pool ID (cognito mode) |
| `azure_auth_mode` | `certificate`, `secret`, or empty (public client) |
| `client_certificate_path` | Path to PEM certificate (certificate mode) |
| `client_certificate_key_path` | Path to PEM private key (certificate mode) |

## Architecture

```
cmd/credential-process/   Entry point and orchestration
internal/
  azure/       Azure AD confidential client (JWT assertion, secret management)
  config/      Config file loading and profile management
  federation/  AWS credential exchange (STS + Cognito)
  jwt/         JWT decode (no verification)
  oidc/        OAuth2 authorization code flow with PKCE
  portlock/    Port-based locking to prevent concurrent auth
  provider/    OIDC provider detection from domain
  storage/     Credential caching (keyring + session file)
```
